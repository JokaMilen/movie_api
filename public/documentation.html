Bussiness logic URL HTTP Method Query parameters Request body data format Response body data format
Return a list of all movies /movies GET None None A JSON object hoding data about all movies
Return data about single movie by title /movies/[title] GET /movies/:title None A JSON object holding data for a single movie by title, containing description, genre, director, image and if featured.
Exampe:
{ "title":"Mystic River",
"description":"The lives of three men who were childhood friends are shattered when one of them has a family tragedy.",
"genre":"Crime",
"director":"Clint Eastwood",
"image":"mysticriver.img",
"featured": yes }
Return data about genre by title /movies/genre/[name] GET /movies/genre/:name None A JSON object holding data about genre by name
Return data about director /movies/directors/[name] GET /movies/directors/:name None A JSON object holding data about director (bio, birth year, death year)
Allow new users to register /user POST None A JSON object holding data about the user to add:
{ username: string required, email: string required,}
 A JSON object holding data about the user that was added, including ID
Allow users to update their user info (username) /users/[username] PUT /users/:username None A text message indicating username was updated
Allow users to add a movie to their list of favorites /users/[username]/favorites/[movieID] POST /users/username/favorites/:movieID None A text message indicating the movie was added to favorites:
{ "movieID": "",}
Allow users to remove a movie from their list of favorites /users/[username]/favorites/[movieID] DELETE /users/username/favorites/:movieID None A text message the movie was removed from favorites
Allow existing users to deregister /users/[username] DELETE /users/:username None A text message indicating the user was removed
